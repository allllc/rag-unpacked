# 01 — Intro to RAG

The canonical RAG pattern: chunk a corpus, embed the chunks, store them in a vector database, retrieve the most similar chunks at query time, and hand them to an LLM as context.

This module walks through that pipeline end-to-end using **Pinecone** as the vector store.
