# 02 — Graph RAG

Retrieval over a knowledge graph instead of a flat vector index. Uses **Neo4j** for the graph and **LangChain** to orchestrate retrieval and generation.

Graph RAG shines when the answer depends on *relationships* between entities — multi-hop questions, entity disambiguation, and domains with rich structure (supply chains, org charts, biomedical data).
